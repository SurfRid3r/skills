# 文章审核工作流程

## 审核流程

当用户请求审核文章时，按以下步骤进行：

### 1. 理解需求
- 确认用户需要审核的文章内容（文本或文件路径）
- 了解用户的目标读者和写作目的（如果已知）
- 询问审核的侧重点（全面审核 or 特定方面）

### 2. 执行审核
读取 [checklist.md](checklist.md) 检查清单，按照三个部分逐一审核：

**第一部分：目标与读者**
- 检查中心思想是否清晰
- 检查目标读者定位是否准确
- 检查标题和摘要的吸引力

**第二部分：结构与逻辑**
- 检查层次结构是否清晰（标题-章节-段落）
- 检查逻辑关系是否明确（时间/流程/空间/主次）
- 检查论证是否流畅无跳跃

**第三部分：内容与表达**
- 检查行文节奏（背景-冲突-解答）
- 检查画面感（场景化、比喻、具象化）
- 检查术语解释

### 3. 输出改进建议

以结构化方式输出审核结果：

```
## 文章审核报告

### 总体评价
[一句话总结文章整体质量]

### 具体改进建议

#### 🎯 目标与读者
- [问题点描述]
  - 当前问题：[具体描述]
  - 改进建议：[具体建议]
  - 示例：[修改前] → [修改后]

#### 📐 结构与逻辑
- [问题点描述]
  - 当前问题：[具体描述]
  - 改进建议：[具体建议]
  - 示例：[修改前] → [修改后]

#### ✍️ 内容与表达
- [问题点描述]
  - 当前问题：[具体描述]
  - 改进建议：[具体建议]
  - 示例：[修改前] → [修改后]

### 优点总结
[列出文章做得好的地方]

### 优先级建议
1. [高优先级] - [具体问题]
2. [中优先级] - [具体问题]
3. [低优先级] - [具体问题]
```

## 审核原则

1. **具体化**：避免空泛建议，提供可操作的修改方向
2. **举例说明**：用"修改前 → 修改后"的例子展示改进方法
3. **平衡反馈**：既指出问题，也肯定优点
4. **优先级分明**：帮助用户聚焦最关键的改进点
5. **目标导向**：所有建议都服务于"既专业又好读"的核心目标
